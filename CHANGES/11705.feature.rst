Add opt-in :class:`~aiohttp.TCPConnector` parameter ``use_truststore=True`` for OS-native trust store integration via the optional `truststore <https://truststore.readthedocs.io/>`_ library. When enabled, :class:`~aiohttp.TCPConnector` builds its verified SSL context with :class:`!truststore.SSLContext`, which delegates certificate verification to the macOS Keychain or Windows certificate stores. This resolves common ``CERTIFICATE_VERIFY_FAILED`` errors for users behind enterprise TLS-intercepting proxies whose root CA lives in the OS trust store. The default behaviour is unchanged. Install the optional dependency with ``pip install aiohttp[truststore]`` -- by :user:`Krishnachaitanyakc`.
